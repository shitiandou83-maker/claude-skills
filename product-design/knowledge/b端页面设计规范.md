# DongDesign B端页面设计规范

## 一、设计理念

本规范为京东B端dongDesign 2.0版本的基础视觉设定，核心设计主张为"简洁、高效、一致、共生"，通过统一的视觉元素（色彩、图标、文字、圆角、动效）实现品牌一致性与体验流畅性，为中心中后台产品打造具备竞争力的产品体验。

## 二、常用组件及基础规范

### 1. 容器类

#### ✅ 卡片 Card — 页面级卡片

**使用场景**：页面级卡片是B端后台中除工作台首页仪表盘外所有内容页面的标准容器，用于包裹页面标题及所有主体内容，且页面标题及所有主体内容需共同包裹在同一个卡片内。它统一应用于列表页、表单页、详情页、配置页等常规场景，通过一致的圆角、边距与布局规则，为页面内容提供清晰的结构边界和视觉聚焦，确保用户在不同页面间切换时获得连贯的认知与操作体验。

此规范不适用于工作台首页（允许自由布局）及特殊全屏弹窗等定制化场景。卡片需占据整个页面内容区域（除导航、面包屑）。

**属性设置**：
- 页面级卡片不需要展示边框和投影，即 bordered 的值为 -，shadow 的值为 never
- （若用户提供的 prompt 中未进行特殊说明，则此处提到的属性设置必须遵守，未提到的属性需使用组件的默认值）

**卡片内边距**：
- 左右内边距为 24px
- 上方内边距为 20px
- 下方内边距根据内容自动延伸至浏览器窗口底部（即下方不留额外内边距，内容到底部）

**圆角**：
- 左上角和右上角为 8px 圆角
- 左下角和右下角为 0px（即直角）

**边框**：无边框，bordered 的值为 -

**阴影**：无阴影，shadow 的值为 never

**内部结构**：
- 卡片内部从上到下分页面标题和内容区域两部分
- 页面标题与内容区域之间的间距为 20px
- 内容区可按照不同业务场景定义展示内容，可以包含任何页面内容（如表单、表格、图表等）
- 卡片底部与浏览器窗口底部接触，内容区域根据内容高度自动扩展

**页面标题**：位于卡片顶部，与内容区域左侧对齐，为 H1 级别的标题样式，字号为 20px，字重为 medium

**加载状态**：当页面内容加载时，卡片应显示加载状态（如骨架屏）

**错误状态**：当内容加载失败时，卡片内应显示错误提示

#### ✅ 内容卡片

**使用场景**：放在普通卡片（页面级卡片）内部，用于进一步组织、分隔、强调子内容布局或实现子内容区域内内容的独立滚动的次级容器

**属性设置**：
- 内容卡片不需要展示边框和投影，即 bordered 的值为 -，shadow 的值为 never，bg-color 为 fill-100

**卡片内边距**：
- 左右内边距为 20px
- 上方内边距为 16px
- 卡片高度需包裹内容，可设置最大展示高度并实现区域内内容的独立滚动，宽度按需撑满父容器或适应内容宽度

**圆角**：4px

**边框**：无边框，bordered 的值为 -

**阴影**：无阴影，shadow 的值为 never

**背景色**：fill-100

**加载状态**：当页面内容加载时，卡片应显示加载状态（如骨架屏）

**错误状态**：当内容加载失败时，卡片内应显示错误提示

#### ✅ 数据指标卡

**使用场景**：主要用于B端后台的仪表盘、概览页面或数据摘要区域，以紧凑、可视化的形式展示关键业务指标和核心统计数据（如销售额、用户增长、完成率、系统状态等），帮助用户快速获取业务洞察、监控实时性能或识别趋势。它们通常以网格或行列布局呈现，突出数字和变化，支持快速决策和状态跟踪，适用于需要高频查看或强调数据驱动的场景，需要提供数据下钻时可支持hover和选中效果

**样式规范**：
- 数据指标卡片一般情况下需嵌套在卡片容器内并展示边框，bg-color 为 #ffffff
- 可支持选中，选中时边框高亮，bordered 的值变为主题色值

**卡片内边距**：
- 左右内边距为 16px
- 上方内边距为 12px
- 卡片高度需包裹内容，宽度按需撑满父容器或适应内容宽度

**圆角**：4px

**数据名称**：字号 14px，加粗，字重为 medium，色值 text-300

**数据值**：使用京东正黑字体，字号 24px，字重为 regular，色值 text-400

**子指标名称**：字号 12px，加粗，字重为 medium，色值 text-200

**子指标数据值**：字号 12px，加粗，字重为 medium，需用颜色表示涨跌时参照红跌绿涨规则，无需用颜色表示涨跌时色值为 text-300

**加载状态**：当卡片内数据加载时，展示数据名称，具体数值展示加载中，色值为 text-100

**错误状态**：当卡片内数据加载时，展示数据名称，具体数值展示加载失败，色值为 error-normal

#### ✅ 气泡卡片 Popover

**使用场景**：鼠标悬停、聚焦或点击在某个目标件时，弹出的气泡式的卡片浮层。可以用于对目标元素进行说明，或展示目标元素的相关扩展项。

**样式规范**：需设置 arrowPointAtCenter，且箭头将指向目标元素的中心。

#### ✅ 抽屉 Drawer

**使用场景**：用于需在保持当前页面上下文不中断的前提下的进行深层次或扩展性的查看或操作，可提供比弹窗更大的横向空间，适合展示复杂内容；同时其滑入动画模拟了"侧边进入-返回"的导航体验，暗示这是一个临时层，而非中断性对话。它特别适合在需要用户参照背景内容（如主列表或数据）时进行关联操作，平衡了操作聚焦与上下文保留。

#### ✅ 弹窗

**页面级弹窗 Modal**：用于需要用户立即处理并绝对聚焦的独立任务，通过遮罩阻断背景交互，强制用户完成或明确放弃当前操作。典型场景包括：进行高风险确认（如删除）、完成一个封闭的表单流程、处理需全屏注意的复杂信息，或响应必须知晓的系统级通知。它与抽屉的关键区别在于其中断性和独占性，适用于必须明确"开始-完成/取消"闭环的关键环节。

**气泡弹窗**：用于在触发元素旁提供即用即走的轻量级辅助信息或操作（如快捷菜单、简短提示、简易选择），其特点是非中断性（无蒙层，不阻断背景交互）与强上下文关联；相较于抽屉适合需要参照背景的扩展性操作，或页面级弹窗用于强制聚焦的独立关键任务，气泡弹窗更侧重于对当前操作的瞬时辅助与局部补充，是三种弹窗形式中最为轻量和即时的一种。

**气泡确认框 PopConfirm**：附着于操作元素（如按钮或链接）旁的轻量级二次确认控件，用于对低风险但需防误操作的行为（如删除单项、取消编辑等）进行即时确认；它通过简短的问询和明确的"确定/取消"按钮，在不阻断背景操作的前提下防止点击失误，比页面级弹窗的强制确认更轻量，比普通气泡弹窗更具行动导向，是平衡操作安全性与流程流畅性的高效设计。

**引导弹窗 Guide**：在用户首次接触新产品、新功能或复杂界面时，通过分步高亮并解释关键界面元素，引导用户快速理解核心操作流程和功能布局。它适用于以下情况：
- 新用户首次进入产品：提供产品核心价值与基本操作路径的快速导览，降低学习门槛
- 新功能上线或重大改版：主动突出新增或重构的功能模块，帮助老用户平滑过渡
- 复杂流程或隐藏功能的教学：逐步演示多步骤操作或不易发现的交互，提升功能发现率和操作效率
- 关键业务引导：在特定场景（如首次创建内容、完成关键设置）中提示最佳实践或注意事项

#### ✅ 表格容器 Table

**使用场景**：适用于任何需要用户对数据集进行系统性查看、分析、筛选或编辑的后台页面。

**属性设置**：
- 表格容器内由不同的表格列构成，并在只在外边框有 1px 的描边，描边色值为 line-100
- 每行单元格有下 1px 边框
- 容器的圆角半径为 4px

**卡片内边距**：容器高度需适应内容列高度，可设置最大展示高度并实现区域内内容的独立滚动，宽度撑满父级容器宽度

**圆角**：4px

**边框**：有边框，bordered 的值为 true，Stroke weight 为 1px，色值为 line-100

**加载状态**：当表格内数据内容加载时，表头直接展示，内容区域显加载失败的提示文案，色值为 text-200

**空状态**：当表格内数据内容为空时，表头直接展示，内容区域显数据为空相关的提示文案，色值为 text-200

**加载失败**：当表格内数据内容加载失败时，表头直接展示，内容区域显加载失败的提示文案，色值为 text-200

### 2. ✅ 文本类

#### 标题

**页面标题**：页面标题即页面级卡片容器的标题，字号使用 18px，字重为 Semibold，色值 text-400

**分段标题**：页面内内容需要进行分组展示时，分组内容的标题，字号使用 16px，字重为 Semibold，色值 text-400

**表格列标题**：表格列内的表头，字号使用 14px，字重为 medium，色值 text-400

**表单标签 Label**：表单控件标签，字号使用 14px，字重为 regular，色值 text-300

**弹窗标题**：字号使用 16px，字重为 Semibold，色值 text-400

**抽屉标题**：字号使用 18px，字重为 Semibold，色值 text-400

#### 正文

**加强文案**：表格中的商品标题，或其他需要强调展示的内容，字号使用 14px，字重为 medium，色值 text-400

**正文文案**：常规描述文案，表单项已填入的内容，字号使用 14px，字重为 regular，色值 text-400

**次要文案**：辅助性描述文案，字号使用 12px，字重为 regular，色值 text-200

**注释文案**：字号跟注释说明的内容字号保持一致，字重为 regular，色值 text-200

**表单暗文 hint**：字号 14px，字重为 regular，色值 text-100

### 3. ✅ 操作按钮类

#### 主要操作按钮

**使用场景**：操作按钮区的主按钮，有且只有一个

**属性设置**：主要操作按钮为主题色填充样式，且须展示前置图标
- **按钮类型**：type 值为 primary
- **icon**：主要操作按钮需要展示前置图标，可以设置 icon 属性，或者直接在 Button 内使用 Icon 组件
- **间距**：按钮之间间距为 8px

#### 次要操作按钮

**使用场景**：查询表单中的重置操作，以及操作按钮区存在多个操作按钮时，跟随在主要操作按钮右侧的操作按钮使用次要操作按钮样式

**属性设置**：次要操作按钮为次要线性样式，且不能占示前置图标
- button 参数 ghost type 取值为 Plain
- icon：不需要展示图标
- 间距：按钮之间间距为 8px

#### 查询按钮

**使用场景**：查询表单区的触发查询动作的操作按钮，一般跟重置按钮同时出现

**属性设置**：查询按钮为主题色填充样式，不示前置图标，重置按钮为次要线性样式，不展示前置图标
- button 参数 ghost type 取值为 primary
- icon：不需要展示图标
- 间距：按钮之间间距为 8px

#### 重置按钮

**使用场景**：查询表单区的重置筛选条件的操作按钮，必须跟查询按钮同时出现并位于查询按钮右侧

**属性设置**：重置按钮为次要线性样式，不展示前置图标
- 按钮类型：ghost type 值为 plain
- icon：不需要展示图标
- 间距：按钮之间间距为 8px

#### 扩展操作按钮

**使用场景**：层级或重要程度低于次要操作按钮和重置按钮的操作，位于次要操作按钮或重置按钮的右侧，如查询表单中按钮组里的展开、收起、导出等操作

**属性设置**：扩展操作按钮为 Text 样式，不展示前置图标，若需表达展开收起操作时，续搭配后置图标使用
- 按钮类型：ghost type 值为 text
- icon：不需要展示图标，若需表达展开收起操作时，续搭配后置图标使用可以设置 icon 属性，或者直接在 Button 内使用 Icon 组件
- 间距：按钮之间间距为 8px

#### 表格内的操作按钮

**使用场景**：表格操作列内的操作按钮

**属性设置**：主要操作按钮为链接样式，不展示前置图标
- 按钮类型：link type 值为 primary
- icon：不需要展示图标
- 间距：按钮之间间距为 8px

#### 表格配置按钮

**使用场景**：控制表格展示的操作，一般有且只有全屏、刷新、紧凑、表格设置四个类型的操作

**属性设置**：表格配置按钮使用带前置图标的文本样式
- 按钮类型：type 值为 text
- icon：不需要展示图标
- 间距：按钮之间间距为 8px

### 4. 导航框架

**使用场景**：B端页面生成时的导航框架，有 ProMenu、ProSingleMenu 单列菜单和 ProTopMenu 顶部菜单，ProTopMenu 必须存在，默认使用 ProSingleMenu 和 ProTopMenu 组合。
